[SCENARIO 1]
description: An agent claiming to be from the state electricity board (like [STATE_ELECTRICITY_BOARD]) calls, stating that the customer's power will be disconnected at 9 PM tonight due to a pending bill from last month. They create extreme urgency and instruct the user to immediately pay a fine to a personal [UPI_APP] number to prevent the disconnection.
case_type: Clear Fraud
platform_type: utility_provider
context: Fake electricity bill payment with disconnection threat
fraud_technique: Urgency creation with payment to personal UPI
policy:
Rule A: Utility companies never ask for payments to be made to personal UPI numbers or bank accounts.
Rule B: Official disconnection notices are sent via post or official SMS, providing at least 15 days of notice, not a few hours.

[SCENARIO 2]
description: A user receives a call from someone claiming to be from their mobile carrier ([MOBILE_CARRIER]). The agent states the user's SIM card will be blocked in two hours due to an incomplete KYC verification. To fix this, the agent asks the user to install a "KYC Verification App," which is actually a remote access tool like AnyDesk.
case_type: Clear Fraud
platform_type: telecom_provide
context: SIM card blocking threat with remote access request
fraud_technique: Remote access software installation for KYC
policy:
Rule A: Mobile carriers never ask customers to install third-party apps for KYC verification.
Rule B: KYC updates are done at official stores or through the official carrier app, initiated by the customer.

Rule B: Government subsidies are managed through official portals, not via phone calls asking for OTPs.

Part 4: Subtle Fraud (5 Scenarios)

[SCENARIO 26]
description: An agent from [MOBILE_CARRIER] calls a user to "confirm their address" for a new SIM card. After the user confirms, the agent says, "Thank you. As a final verification step, please tell me your father's full name, which we have on record." This is an attempt to phish for a common security question answer.
case_type: Subtle Fraud
platform_type: telecom_provider
context: Address verification with security answer phishing
fraud_technique: Security question answer theft
policy:
Rule A: Agents must not ask for information commonly used as security question answers (e.g., father's name, mother's maiden name, place of birth).
Rule B: Verification should be done using less sensitive data points.

[SCENARIO 27]
description: A scammer claiming to be from the [STATE_ELECTRICITY_BOARD] uses information from a data breach. They call and say, "I see your last bill was ₹1,250. There was a calculation error, and you are due a refund. To process it, I need you to confirm your full bank account number."
case_type: Subtle Fraud
platform_type: utility_provider
context: Data breach exploitation with refund pretext
fraud_technique: Phishing for bank account number
policy:
Rule A: Utility companies process refunds as credits on the next bill or to the original payment source; they do not ask for bank account numbers over the phone.
Rule B: Using specific, breached information to appear legitimate is a common social engineering tactic.
